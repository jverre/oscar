# Oscar Chat Setup Instructions

## Prerequisites

1. Node.js 18+ installed
2. A Google OAuth application (for authentication)
3. A Convex account

## Environment Setup

### 1. Configure Hosts File

Add these entries to your hosts file:

**macOS/Linux**: Edit `/etc/hosts`
**Windows**: Edit `C:\Windows\System32\drivers\etc\hosts`

```bash
# Oscar Chat local development
127.0.0.1 oscar-chat.local
127.0.0.1 acme.oscar-chat.local
127.0.0.1 test-org.oscar-chat.local
127.0.0.1 demo.oscar-chat.local
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Set up Convex

```bash
# Initialize Convex (follow prompts to create new project)
npx convex dev
```

This will:
- Create a new Convex project
- Generate `NEXT_PUBLIC_CONVEX_URL` and `CONVEX_DEPLOYMENT` values
- Start the Convex dev server

### 4. Configure Google OAuth

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select existing
3. Enable Google+ API
4. Create OAuth 2.0 credentials
5. Add authorized redirect URIs:
   - `http://oscar-chat.local:3000/api/auth/callback/google`
   - Add more subdomains as needed

### 5. Update Environment Variables

Edit `.env.local` with your values:

```env
# Convex (generated by convex dev)
NEXT_PUBLIC_CONVEX_URL=https://your-convex-url
CONVEX_DEPLOYMENT=your-deployment-name

# Auth
AUTH_SECRET=your-generated-secret-here
AUTH_GOOGLE_ID=your-google-client-id
AUTH_GOOGLE_SECRET=your-google-client-secret

# App
NEXT_PUBLIC_APP_URL=http://oscar-chat.local:3000
```

## Running the Application

### Terminal 1: Convex
```bash
npx convex dev
```

### Terminal 2: Next.js
```bash
npm run dev
```

## Testing the Flow

1. **Main Domain**: Visit `http://oscar-chat.local:3000`
   - Should redirect to `/auth/signin`

2. **Sign In**: Click "Continue with Google"
   - Complete OAuth flow
   - Should show organization creation form

3. **Create Organization**: 
   - Enter organization name: "Acme Corp"
   - Enter subdomain: "acme"
   - Click "Create Organization"
   - Should redirect to `http://acme.oscar-chat.local:3000`

4. **Organization Access**: 
   - Visit `http://acme.oscar-chat.local:3000`
   - Should show organization dashboard

## Troubleshooting

### Hosts File Issues
- On macOS, you may need to flush DNS: `sudo dscacheutil -flushcache`
- On Windows, restart browser or run `ipconfig /flushdns`

### Convex Issues
- Make sure `npx convex dev` is running
- Check `.env.local` has correct Convex URLs

### OAuth Issues
- Verify redirect URIs in Google Console match your local domains
- Check `AUTH_GOOGLE_ID` and `AUTH_GOOGLE_SECRET` are correct

### Subdomain Issues
- Ensure middleware is running (check Next.js console)
- Verify hosts file entries are correct
- Try accessing `http://oscar-chat.local:3000/auth/signin` directly

## Production Deployment

When ready for production:

1. Update Google OAuth redirect URIs to production domains
2. Deploy to Vercel with Convex production deployment
3. Configure DNS for wildcard subdomains
4. Update environment variables in production
